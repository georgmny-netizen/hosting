Final Architecture · SEO + UX Control Map
One system, three levels: navigation → action → decision.
The interface is now organized as a procurement decision cockpit: global header for discovery, command rail for controlled actions, Executive layer for approve / condition / reject, and SEO-readable sections for product, compliance, logistics and RFQ intent.
Mitsubishi / Itochu-style desk hierarchy

UX spatial hierarchy

Level 01Header = discovery

SUPPLY / PRODUCT / DEAL / AUDIT remain compact, semantic and crawlable. User always knows where to go.

Level 02Command rail = action

Executive Overview is the primary control. Data Room, Legal, Supply Chain, TDS and pricing actions remain one-click.

Level 03Executive = decision

RFQ evidence, risk, route, margin and hard-gate status are converted into approve / condition / reject.

UX rule: no new floating buttons. Every control must belong to one of three layers: discover, act, or decide.
