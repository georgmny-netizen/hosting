## 📝 Запись 1: Настройка хостинга и интеграция GitHub Pages
**Цель:** Перенос сайта из черновой папки `c:\tmp\` в чистовую директорию `c:\tmp\hosting\` и настройка бесплатного облачного хостинга.

* **Действие:** Инициализация локального Git-репозитория, подключение к приватному репозиторию GitHub, настройка GitHub Pages.
* **Ошибка (IDE):** При попытке сделать первый Commit редактор выдавал красную ошибку: `Author identity unknown. Omit --global to set the identity only in this repository.`
* **Причина:** VS Code в текущей среде Windows по какой-то причине игнорировал глобальные настройки Git (`--global`), прописанные ранее в терминале.
* **Решение:** Проблема решена путем принудительной записи данных автора (name, email) **локально** в конфиг конкретного репозитория `hosting` с использованием абсолютного пути к исполняемому файлу `git.exe`. Коммит успешно прошел.

## 📝 Запись 2: Баг с авто-переводом (Тайваньский и Китайский)
**Цель:** Полная локализация сайта на 4 языка (KO, JA, CN, TW) без засорения основного HTML-файла.

* **Действие:** Запуск Python-скрипта `translate_site.py` для массового перевода английского текста из `index.html` в JSON-файлы.
* **Ошибка:** Большие блоки текста (например, "Live Batch Availability" или характеристики "Dubai Stock") остались на английском. Анализ показал, что в китайских файлах более 55% символов — латиница.
* **Причина:** Скрипт отправлял слишком много параллельных запросов и попал под анти-спам блокировку (Rate Limit) от Google Translate. Встроенный в скрипт обработчик ошибок (`except`) молча возвращал оригинальный английский текст, чтобы программа не вылетала.
* **Решение:** 
    1. Написан умный Python-скрипт допереводчик с увеличенными паузами (`time.sleep`), который просканировал все файлы и доперевел "застрявшие" фразы, игнорируя при этом системные слова вроде `[SYS-LOG]`.
    2. Самые важные и объемные блоки (характеристики продукта, навигационное меню) переведены **вручную** с использованием профессиональной B2B-терминологии.

## 📝 Запись 3: Ошибка навигации визуального агента
**Цель:** Визуально просмотреть сайт в браузере на тайваньском языке для вычитки интерфейса.

* **Действие:** Запуск внутреннего браузерного агента с задачей зайти на сайт, переключить язык на TW и пролистать страницу в поиске оставшихся английских кнопок.
* **Ошибка:** Агент не смог загрузить страницу. В браузере повисла ошибка `chrome-error://chromewebdata/`.
* **Причина (Человеческий фактор ИИ):** Агенту по ошибке была передана устаревшая ссылка на черновик (`http://localhost:8000/CANONICAL.html`), в то время как чистовой сайт уже был перенесен в файл `index.html`.
* **Решение:** Выполнение задачи в браузере было отменено. Вместо этого применен метод **прямого анализа кода** — программное сопоставление английского `index.html` и тайваньского `tw.json`. Метод оказался быстрее и точнее: были найдены и переведены все пропущенные кнопки меню (Supply, Product, APAC Desk и т.д.).
