## Аудит структуры и составление детального плана (Фаза 1 подготовка)

Вот что я буду делать для **полного аудита** перед извлечением inline стилей:

---

## Этап 1: Инвентаризация всех `<style>` блоков

### Что я сделаю:
1. **Найду все inline `<style>` блоки** с их ID, позицией и размером
2. **Проанализирую содержимое** каждого блока (какие селекторы, медиа-запросы)
3. **Определю зависимости** между блоками (кто кого переопределяет)

### Команды:
```bash
# Найти все <style> блоки
Select-String -Pattern '<style id="([^"]+)">' -AllMatches

# Подсчитать строки в каждом блоке
# Проверить, есть ли медиа-запросы
# Найти переопределения селекторов
```

### Результат:
Таблица вида:
```
| ID блока | Строка | Размер | Зависит от | Переопределяет | Категория |
|----------|--------|--------|------------|----------------|-----------|
| yyk-v35-final-architecture-blueprint | 5900 | 30 строк | - | - | Architecture |
| yyk-v37-legal-risk-panel | 5931 | 10 строк | v35 | .yyk-arch-grid @media | Legal |
```

## Этап 2: Анализ порядка загрузки и каскада

### Что я сделаю:
1. **Найду все переопределения** (когда один стиль перезаписывает другой)
2. **Проверю специфичность** селекторов (есть ли `!important`, вложенные селекторы)
3. **Определю критический порядок** загрузки (какие блоки ДОЛЖНЫ идти раньше других)

### Команды:
```bash
# Найти все !important
Select-String -Pattern '!important'

# Найти дублирующиеся селекторы
Select-String -Pattern '\.yyk-arch-grid\{' -AllMatches

# Найти медиа-запросы, которые переопределяют базовые стили
Select-String -Pattern '@media.*yyk-arch'
```

### Результат:
Граф зависимостей:
```
base.css (переменные, reset)
  ↓
layout.css (grid-системы)
  ↓
architecture.css (v35, v36)
  ↓
legal-risk.css (v37) ← переопределяет .yyk-arch-grid
```

---

## Этап 3: Проверка связей между HTML и CSS

### Что я сделаю:
1. **Найду все классы в HTML**, которые используются в inline стилях
2. **Проверю, есть ли "сироты"** (классы в CSS, которых нет в HTML)
3. **Найду конфликты** (одинаковые ID в разных блоках)

### Команды:
```bash
# Найти все классы в HTML
Select-String -Pattern 'class="([^"]+)"' -AllMatches

# Найти все селекторы в <style> блоках
Select-String -Pattern '\.[a-z-]+\{' -AllMatches

# Сравнить списки
```

### Результат:
Отчёт о несоответствиях:
```
✅ Используется в HTML и CSS: .yyk-arch-grid, .yyk-arch-card
⚠️  Только в CSS (не используется): .old-legacy-class
❌ Только в HTML (нет стилей): .new-experimental-feature
```

## Этап 5: Сравнение с CANONICAL версией

### Что я сделаю:
1. **Сравню структуру** `<style>` блоков между bad и CANONICAL
2. **Найду различия** (какие блоки есть только в одной версии)
3. **Проверю, есть ли критические отличия** в порядке загрузки

### Команды:
```bash
# Найти все <style> в CANONICAL
Select-String -Path "CANONICAL.html" -Pattern '<style id="([^"]+)">'

# Сравнить списки ID
# Найти уникальные блоки в каждой версии
```

### Результат:
```
Общие блоки: 18

```

## Этап 6: Проверка JavaScript зависимостей

### Что я сделаю:
1. **Найду все обращения к CSS** из JavaScript (изменение классов, стилей)
2. **Проверю, есть ли динамическая загрузка** стилей
3. **Определю, какие стили критичны** для работы JS

### Команды:
```bash
# Найти все classList.add/remove
Select-String -Path "main.js" -Pattern 'classList\.(add|remove|toggle)'

# Найти все style.display =
Select-String -Path "main.js" -Pattern '\.style\.'

# Найти все querySelector с классами из inline стилей
```

### Результат:
```
Критические классы для JS:
  .yyk-arch-grid → используется в resize handler
  .modal.active → используется в modal.js
  .compliance-engine.active → используется в compliance.js

⚠️  Эти стили НЕЛЬЗЯ загружать асинхронно
```
